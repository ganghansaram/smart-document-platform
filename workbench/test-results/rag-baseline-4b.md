## 3. 핵심 문제 분석

### 3-1. 검색 적중률 0% — 근본 원인

현재 검색 인덱스에 로드된 문서가 테스트 질문의 도메인(항공·방산 기술)과 **거의 일치하지 않음**.

반환된 소스를 분류하면:

| 소스 출처 | 출현 빈도 | 관련성 |
|-----------|:---------:|:------:|
| RAG 연구 논문 (연구 방법론, 실험 결과) | 매우 높음 | **무관** |
| SWA_PMS (소프트웨어 설계 문서) | 높음 | **무관** |
| Air Force Activities Report | 중간 | **부분 관련** |
| DRACO UAV 문서 | 중간 | **무관** |
| QA 시나리오 문서 | 중간 | **무관** |

**결론**: 검색 엔진(BM25+FAISS) 자체의 문제가 아니라, **인덱스에 항공 기술 문서가 부족**한 것이 근본 원인. 현재 인덱스는 RAG 연구 보고서, SWA 설계 문서, 미군 활동 보고서가 주를 이룸.

### 3-2. 모델 문제 — 4b의 한계

검색 결과가 무관해도 4b 모델이 보이는 추가적 문제:

1. **문맥 혼동**: RAG 연구 방법론(BM25, FAISS)을 질문 주제(형상관리, 시험 절차)로 오인
2. **환각 생성**: 관련 문서가 없을 때 일반 지식으로 답변 생성 (47% 환각률)
3. **거부 일관성 부족**: 같은 "정보 없음" 상황에서 때로는 거부, 때로는 환각 생성

### 3-3. 개선 방향

| 우선순위 | 개선 항목 | 기대 효과 |
|:--------:|----------|----------|
| **1** | 테스트용 항공 기술 문서 추가 인덱싱 | 검색 적중률 0% → 60%+ |
| **2** | 프롬프트 강화 (문서 없으면 거부 강제) | 환각률 47% → 10% 이하 |
| **3** | 모델 업그레이드 (27b) | 문맥 이해력, 답변 품질 향상 |
| **4** | 검색 파이프라인 고도화 (Phase 2-3) | 복합 질문 대응력 향상 |

> **참고**: 현재 베이스라인은 "검색 인덱스 콘텐츠 부족"이라는 외부 요인이 지배적이므로, Phase 1 수행 전에 **테스트 질문에 대응하는 문서를 인덱스에 추가**하거나, **인덱스 내 문서에 맞는 테스트 질문으로 교체**해야 의미 있는 전후 비교가 가능함.
