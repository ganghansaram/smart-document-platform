## 3. 핵심 발견

### 3-1. 검색은 잘 동작한다

v1에서 검색 적중률 0%였던 것은 **검색 엔진의 문제가 아니라 테스트 질문과 인덱스 콘텐츠의 불일치**가 원인이었음이 확인됨. 적절한 질문으로 교체하자 적중률 97%로 상승.

- BM25+FAISS 하이브리드 검색은 의도대로 동작
- 5개 소스 모두 관련 문서에서 반환됨 (대부분 정확한 문서/섹션)

### 3-2. 4b 모델의 강점과 약점

**강점** (Tier 1에서 두드러짐):
- 단순 사실 추출 능력 우수 (정확도 4.4/5)
- 수치 인용 정확 ($558.6M, α=0.2 등)
- 한국어 답변 품질 양호
- 환각 억제력 상당히 좋음 (Tier 1-2에서 환각 0%)

**약점** (Tier 2-3에서 나타남):
- **수치 세부 오류**: Top-k=5(실제 8), Threshold=0.25(실제 0.30) — 근사값으로 답변
- **불완전 종합**: 7개 UC 중 5개만 나열, Block 1-3 미포함
- **크로스 문서 혼입**: PLM DD-03 질문에 DRACO DD-03 소스 혼입 (T3-3)
- **추론 깊이 한계**: "상세 내용 미포함" 자백 (T3-5), What-if 분석 회피 (T3-2)

### 3-3. 개선 여지 분석

| 문제 유형 | 해당 문항 | 개선 방안 | 기대 Phase |
|-----------|----------|----------|:----------:|
| 수치 세부 오류 | T2-3, T2-4 | 27b 모델의 더 정밀한 독해력 | Phase 1 |
| 불완전 종합 | T2-4, T2-5 | 컨텍스트 확장 (8000→16000자) | Phase 2 |
| 크로스 문서 혼입 | T3-3 | 메타데이터 필터 (문서별 분리) | Phase 2 |
| 추론 깊이 부족 | T3-2, T3-5 | Agent 루프 (반복 추론) | Phase 3 |
| 카테고리별 분석 부족 | T3-4 | 쿼리 분해 (연도별 서브쿼리) | Phase 2 |
