# 텍스트 선택 AI 액션 메뉴 — 설계안

## 개요

원문 PDF에서 텍스트 드래그 시 등장하는 단일 "마킹" 버튼을 **[마킹] [번역] [요약]** 3버튼 그룹으로 확장.
선택 텍스트에 대해 Ollama LLM을 활용한 즉석 번역/요약 기능 제공.

### 업계 레퍼런스

| 제품 | 텍스트 선택 시 액션 |
|------|-------------------|
| Kindle / Apple Books | 하이라이트, 메모, 사전, 번역, 공유 |
| Notion | AI 요약/번역/설명, 코멘트, 링크 |
| Arc Browser | "Ask AI" |
| macOS 시스템 | 찾기, 번역, 사전 |

---

## 1. 액션 바 (3버튼)

현재 "마킹" 단일 버튼 → 3버튼 그룹으로 확장.

```
드래그 완료 시 등장:
┌──────────────────────────────┐
│  ✏️ 마킹  │  🌐 번역  │  📝 요약  │
└──────────────────────────────┘
```

- **위치**: 현재와 동일 — 선택 영역의 끝점(mouseup) 근처
- **형태**: 가로 버튼 그룹, 둥근 필(pill) 형태
- **구분선**: 버튼 사이 1px 세로선 (같은 그룹이라는 인지)
- 클릭 영역 밖 클릭 또는 ESC → 액션 바 닫힘 (기존 동작 유지)

---

## 2. 결과 팝오버

번역/요약 클릭 시 **액션 바가 결과 팝오버로 교체**됨 (같은 위치).

```
┌─────────────────────────────────┐
│  🌐 번역                    ✕  │  ← 헤더 (액션 타입 + 닫기)
├─────────────────────────────────┤
│                                 │
│  번역/요약 결과 텍스트가        │  ← 본문 (max-height: 240px, 스크롤)
│  여기에 표시됩니다.             │
│                                 │
├─────────────────────────────────┤
│           [복사]  [마킹+메모]   │  ← 하단 액션
└─────────────────────────────────┘
```

- **너비**: 280~320px
- **max-height**: 300px (본문 스크롤)
- **위치**: 선택 영역 바로 아래. 하단 공간 부족 시 위로 뒤집기
- **닫기**: ✕ 버튼, 외부 클릭, ESC

### 하단 액션 버튼

| 버튼 | 동작 |
|------|------|
| **복사** | 결과 텍스트를 클립보드에 복사 → "복사됨 ✓" 피드백 |
| **마킹+메모** | 선택 영역을 형광펜 마킹 + AI 결과를 메모로 저장 |

"마킹+메모"는 기존 마킹 생성 + memo 필드에 AI 결과를 자동 기입.
별도 API 호출 없이 기존 POST annotations에 memo를 포함하면 됨.

## 3. 로딩/대기 상태

**방식: 즉시 팝오버 + 스켈레톤 → 완료 시 텍스트 교체**

```
클릭 직후 (0ms):
┌─────────────────────────────────┐
│  🌐 번역                    ✕  │
├─────────────────────────────────┤
│  ████████████████████  (shimmer)│  ← 스켈레톤 3줄
│  ██████████████  (shimmer)      │     (pulse 애니메이션)
│  ████████████████████  (shimmer)│
├─────────────────────────────────┤
│           [복사]  [마킹+메모]   │  ← 비활성(disabled) 상태
└─────────────────────────────────┘

응답 수신 후:
┌─────────────────────────────────┐
│  🌐 번역                    ✕  │
├─────────────────────────────────┤
│  이 연구는 대규모 언어 모델의   │  ← fade-in 전환
│  성능을 평가하기 위한 새로운    │
│  벤치마크를 제안한다.           │
├─────────────────────────────────┤
│           [복사]  [마킹+메모]   │  ← 활성화
└─────────────────────────────────┘
```

### 로딩 방식 비교

| 방식 | 장단점 |
|------|--------|
| 스피너 | 크기 대비 정보량 없음, "멈춘 건가?" 불안감 |
| "번역 중..." 텍스트 | 단순하지만 밋밋함 |
| **스켈레톤 shimmer** | 결과물의 형태를 미리 암시, 체감 대기시간 감소 (업계 표준) |
| 스트리밍 | 최고의 UX이지만 SSE 구현 필요 → v2 고려 |

> **v2 개선안**: Ollama streaming API + FastAPI `StreamingResponse`로 글자 단위 출력.
> 현 설계에서 팝오버 구조를 바꿀 필요 없이, 본문 영역만 스트리밍으로 교체하면 됨.

## 4. 백엔드 API

단일 엔드포인트로 통합 (액션 타입만 다름):

```
POST /api/translator/ai/selection
Authorization: 쿠키 (viewer 이상)

Request:
{
  "text": "선택된 원문 텍스트",
  "action": "translate" | "summarize",
  "model": "gemma3:4b"           // 현재 선택된 모델 사용
}

Response (200):
{
  "result": "AI 응답 텍스트"
}

Error:
- 400: text가 비어있거나 너무 김
- 502: Ollama 연결 실패
- 504: 타임아웃
```

### 백엔드 처리 흐름

```
1. text 길이 검증 (최소 1자, 최대 ~3000자)
2. action에 따른 시스템 프롬프트 선택
3. Ollama /api/generate 호출 (동기 대기)
4. 결과 반환
```

### 프롬프트 설계

| 액션 | 시스템 프롬프트 (요약) |
|------|----------------------|
| `translate` | "다음 텍스트를 한국어로 자연스럽게 번역하세요. 원문의 기술 용어는 괄호 안에 영문을 병기하세요. 번역문만 출력하세요." |
| `summarize` | "다음 텍스트를 한국어로 3문장 이내로 요약하세요. 핵심 논점만 간결하게 전달하세요. 요약문만 출력하세요." |

- 프롬프트는 `config.py`에 설정값으로 관리 (admin 설정에서 커스터마이즈 가능)
- **타임아웃**: 30초 (페이지 번역 5분과 달리, 선택 텍스트는 짧으므로)

---

## 5. 상호작용 흐름

```
사용자: 좌측 원문에서 텍스트 드래그
    ↓
액션 바 등장: [마킹] [번역] [요약]
    ↓
┌─── 마킹 클릭 ──→ 기존 동작 (형광펜 즉시 생성)
│
├─── 번역 클릭 ──→ 액션 바 → 결과 팝오버(스켈레톤)
│                    ↓
│                  API 호출 → 응답 수신
│                    ↓
│                  결과 표시 → [복사] or [마킹+메모]
│                    ↓
│              ┌─ 복사 → 클립보드 + "복사됨 ✓"
│              └─ 마킹+메모 → 형광펜 생성 + memo에 번역문 저장
│
└─── 요약 클릭 ──→ (번역과 동일 흐름, 프롬프트만 다름)
```

**페이지 이동/줌 시**: 팝오버 즉시 닫힘 (기존 marking popover와 동일 패턴)

## 6. 엣지 케이스

| 상황 | 처리 |
|------|------|
| 선택 텍스트가 1~2단어 | 번역/요약 모두 정상 처리 (짧은 번역은 용어 번역 역할) |
| 선택 텍스트가 매우 긺 (3000자+) | 요청 전 잘라서 전송 + 팝오버에 "(일부만 처리됨)" 표시 |
| Ollama 미실행/연결 실패 | 팝오버 본문에 에러 메시지: "AI 서비스에 연결할 수 없습니다" |
| 응답 지연 (30초 타임아웃) | "응답 시간이 초과되었습니다. 더 짧은 텍스트로 시도해 주세요." |
| 팝오버 열린 상태에서 새로 드래그 | 기존 팝오버 닫기 → 새 액션 바 표시 |
| 결과 팝오버가 화면 밖으로 넘침 | 위/아래 뒤집기 + 좌우 패널 경계 클램핑 |

## 7. 코드 영향 범위

| 영역 | 변경 |
|------|------|
| `translator.html` CSS | 액션 바 3버튼 스타일, 결과 팝오버 스타일, 스켈레톤 애니메이션 |
| `translator.html` JS | 액션 바 → 3버튼 생성, 번역/요약 클릭 핸들러, API 호출, 팝오버 렌더 |
| `backend/api/translator.py` | `POST /ai/selection` 엔드포인트 1개 추가 |
| `backend/services/translator_service.py` | `ai_selection_query()` 함수 — Ollama 호출 |
| `backend/config.py` | `TRANSLATOR_AI_SELECTION_TIMEOUT`, 프롬프트 설정값 |
| 기존 마킹 기능 | **변경 없음** (마킹 버튼은 동일 동작 유지) |

---

## 8. 핵심 설계 포인트

1. 액션 바 → 결과 팝오버 **같은 위치 교체** (화면 이동 없음)
2. **스켈레톤 로딩** → 결과 fade-in (체감 대기 최소화)
3. **[마킹+메모]** 버튼으로 AI 결과를 기존 마킹 시스템에 자연스럽게 연결
