```
2단계  Cross-encoder 리랭킹 → Top 5
3단계  중복 제거 → LLM 전달
```

**좌측 하단 — 구조 보존 인덱싱**:
- 테이블 → GFM Markdown, 수식 → LaTeX, 병합셀 → KV 형식
- 결과: 테이블/수식 기반 질문·답변 **가능**

**우측 하단 — DOCX 변환 + 렌더링 최적화**:
- COM 전처리(장절번호) → python-docx(구조 변환) → OMML→MathML(수식)
- content-visibility:auto로 4MB HTML 부드러운 스크롤 (수렴 알고리즘)

| `DISPLAY_CONFIG.tableStyle` | `bordered` | 테이블 스타일 (bordered/simple/minimal) |

**런타임 설정** — `data/settings.json` (관리자 설정 페이지에서 자동 생성):

- 관리자 웹 GUI에서 변경한 설정은 `data/settings.json`에 저장됩니다
- 백엔드 시작 시 자동 로드 → `config.py` 변수에 오버라이드
- 프론트엔드는 `/api/settings/public` API로 화면 관련 설정을 가져옵니다
- `config.js`는 기본값 역할, `settings.json`이 우선 적용됩니다

### C-4. 인덱스 관리 (CLI)

### C-5. 문제 해결

| 증상 | 확인 사항 | 해결 |
|------|----------|------|
| 백엔드 미연결 | FastAPI 실행 여부 | `python backend/main.py` 재실행, 포트 8000 확인 |
| AI 응답 오류 | Ollama 연결 | `ollama list`로 모델 확인, `OLLAMA_URL` IP 확인 |
| Ollama 연결 실패 | 방화벽 | Linux 11434 포트 개방 |
| 검색 결과 없음 | 인덱스 파일 | `data/search-index.json` 존재 여부, 재인덱싱 실행 |
| 업로드 실패 | 권한/용량 | `contents/` 폴더 쓰기 권한, 파일 500MB 이하 확인 |
| 업로드 변환 실패 (DRM) | COM 전처리 | `config.py`에서 `WORD_COM_PREPROCESS = False` 설정, 또는 수동 전처리 후 업로드 |
| 로그인 불가 | DB 파일 | `data/auth.db` 존재 여부, `create-admin.py` 재실행 |
| 설정 변경 미반영 | 재시작 필요 항목 | Ollama URL/모델, CORS 등은 백엔드 재시작 필요 (설정 페이지에 표시) |
| 로그인 페이지 타이틀 안 바뀜 | 백엔드 연결 | 백엔드가 실행 중이어야 `/api/settings/public`에서 타이틀 로드 |
