## Slide 03 — 1. 개요: 배경과 목적

**제목**: 1. 개요 — 왜 이 플랫폼인가

**좌측 — 기존 문제점** (아이콘 + 텍스트):
- **종이 문서**: 물리적 보관 한계, 검색 불가, 훼손 위험
- **PDF 파일**: 폴더 분산, 전문 검색 불가, 문서 간 이동 불편
- **외부 서비스**: 폐쇄망 사용 불가, 보안 규정 위배
- **영문 기술문서**: 번역 도구 부재, 개인별 수작업 번역

**우측 — 플랫폼이 제공하는 해결**:
- 수천 페이지 기술문서를 **단일 플랫폼**에서 통합 열람
- 키워드 + AI 의미 검색으로 **필요한 정보 즉시 탐색**
- 영문 논문/규격서를 **페이지별 온디맨드 번역**으로 즉시 독해
- 폐쇄망 완전 독립 — **외부 의존성 Zero**

**하단 — 핵심 가치 3가지** (강조 박스):
- **설치 한 번으로 완결** — CDN · API · 프레임워크 의존성 Zero, 폴더 복사만으로 서비스
- **빌드 도구 불필요** — npm, webpack 없이 정적 파일만으로 프론트엔드 구동
- **3단계 역할 기반 접근 제어** — viewer(열람) / editor(편집) / admin(관리) 역할별 기능 분리

## Slide 09 — 3. Explorer 핵심 기능 ① 열람 · 검색

**제목**: 3. Explorer 핵심 기능 — 열람 · 검색

**좌측 — 열람 기능** (6개):
- **트리 메뉴 탐색**: 다단계 계층 메뉴, 현재 위치 자동 하이라이트, 트리 내 검색
- **섹션 목차 (On This Page)**: h1~h6 자동 수집, 스크롤 위치 실시간 추적, 접기/펼치기
- **우측 패널 자동 토글**: 홈→자동 접힘, 문서→자동 펼침 (수동 조작 시 존중)
- **다크/라이트 테마**: 해/달 아이콘 토글, localStorage 유지
- **북마크**: 헤딩 단위 저장, 문서 간 이동
- **키보드 단축키**: `?`(도움말), `←→`(문서 이동), `/`(검색), `H`(홈), `B`(북마크)

**우측 — 검색 기능** (3단계 파이프라인):

```
사용자 검색어
  ├─ 키워드 검색 (BM25, 30%) ─┐
  │                             ├─ RRF 병합 → 리랭킹 → Top 5
  └─ 벡터 검색 (FAISS, 70%) ──┘
```

- **하이브리드 검색**: BM25 키워드 + FAISS 벡터, RRF(Reciprocal Rank Fusion) 병합
- **Cross-encoder 리랭킹**: 15개 후보 → 5개 정밀 재정렬 (bge-reranker-v2-m3)
- **구조 보존 인덱싱**: 테이블→GFM Markdown, 수식→LaTeX — 테이블/수식 기반 질의 가능
- **용어집 통합 검색**: 검색 결과 상단에 항공 용어집 일치 항목 함께 표시

**좌측 — AI 챗봇 (RAG 기반)**:

**우측 — PPT 문서 지원 (Explorer 확장)**:

현재 Explorer는 DOCX/PDF만 업로드 가능. 사내 문서 중 PPT 원본이 다수 존재.

**변환 전략**: 슬라이드 이미지 + 히든 텍스트

```
PPTX 업로드
  ├─ python-pptx: 텍스트/노트 추출
  └─ LibreOffice headless: 슬라이드 → PNG

             <img src="slide_001.png">
             <div class="hidden">텍스트 내용</div>
```

- 시각적 충실도: 슬라이드 이미지 = 원본 그대로
- 검색/RAG: 히든 텍스트 → 기존 인덱싱 파이프라인 그대로 동작

### B-7. AI 텍스트 선택 (번역/요약)

| 동작 | 방법 |
|------|------|
| 텍스트 선택 | 좌측 패널에서 텍스트 드래그 → 액션 바 표시 |
| 즉시 번역 | 액션 바의 "번역" 클릭 → AI 번역 결과 Popover에 표시 |
| 즉시 요약 | 액션 바의 "요약" 클릭 → AI 요약 결과 Popover에 표시 |
| 결과 복사 | Popover 내 "복사" 버튼 |
| 메모로 저장 | Popover 내 "메모로 저장" → 마킹 생성 + AI 결과를 메모에 첨부 |

### B-8. 문서 검색

## 별첨 D — 시스템 관리자 (배포/운영) 매뉴얼

> 대상: 서버 설치 · 배포 · 유지보수를 담당하는 IT 관리자

### D-1. 운영 환경 요구사항

| 항목 | Windows 서버 | Linux GPU 서버 |
|------|-------------|---------------|
| OS | Windows Server 또는 Windows 10+ | Ubuntu / CentOS |
| Runtime | Python 3.11+, JDK 1.8+ | Ollama |
| Service | Tomcat (8080), FastAPI (8000) | Ollama (11434) |
| ML Model | bge-reranker-v2-m3 (로컬, ~500MB) | gemma3:4b (~3GB), bge-m3 (~1.5GB) |
| Storage | ~500MB (플랫폼) + 문서 용량 | ~5GB (모델) |
| 선택 | pywin32 + Microsoft Word (장절번호 COM 전처리) | — |

### D-2. 초기 배포

| 단계 | 명령 / 동작 | 비고 |
|------|------------|------|
| 1 | 프로젝트 폴더를 Tomcat `webapps/ROOT/`에 복사 | 기존 ROOT 폴더 백업 후 교체 |
| 2 | `pip install -r requirements.txt` | Python 패키지 설치 (오프라인: 미리 다운로드) |
| 3 | `python tools/create-admin.py` | 최초 관리자 계정 생성 |
| 4 | Linux: `ollama pull gemma3:4b && ollama pull bge-m3` | 모델 다운로드 (오프라인: 파일 복사) |
| 5 | `js/config.js`에서 서버 IP 수정 | `backendUrl`, `ollamaUrl` |

**Translator 추가 설정**:
- `pdf2zh` (PDFMathTranslate) 설치: `pip install pdf2zh`
- 첫 실행 시 `~/.cache/babeldoc/`에 CMap/폰트/모델 ~500MB 자동 다운로드
- 폐쇄망: 사전에 캐시 파일을 복사하여 배치

### D-3. 서비스 실행 순서

| 순서 | 서버 | 명령 | 포트 |
|------|------|------|------|
| 1 | Linux | `ollama serve` | 11434 |
| 2 | Windows | `python backend/main.py` | 8000 |
| 3 | Windows | Tomcat `startup.bat` | 8080 |
| 4 | — | 브라우저에서 `http://서버IP:8080/` 확인 | — |

### D-4. 설정 파일

**정적 설정** — `js/config.js` (배포 시 1회 수정):

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `AUTH_CONFIG.loginRequired` | `true` | 로그인 필수 여부 |
| `AUTH_CONFIG.backendUrl` | `localhost:8000` | 백엔드 서버 주소 |
| `AI_CONFIG.enabled` | `true` | AI 챗봇 on/off |
| `AI_CONFIG.useBackend` | `true` | 백엔드 RAG 사용 |
| `AI_CONFIG.searchType` | `hybrid` | 검색 방식 |
| `EDITOR_CONFIG.enabled` | `true` | 편집 기능 on/off |
| `UPLOAD_CONFIG.enabled` | `true` | 업로드 기능 on/off |

**런타임 설정** — `data/settings.json` (관리자 설정 페이지에서 자동 생성):
- 관리자 웹 GUI에서 변경한 설정은 `data/settings.json`에 저장
- 백엔드 시작 시 자동 로드 → `config.py` 변수에 오버라이드
- 프론트엔드는 `/api/settings/public` API로 화면 관련 설정을 가져옴
- `config.js`는 기본값 역할, `settings.json`이 우선 적용

### D-5. 인덱스 관리 (CLI)

| 작업 | 명령 |
|------|------|
| 검색 인덱스 재생성 | `python tools/build-search-index.py --inject-ids` |
| 벡터 인덱스 재생성 | `python tools/build-vector-index.py` |
| ID 주입만 | `python tools/build-search-index.py --inject-ids --no-index` |

- 웹 UI 업로드 사용 시 인덱스가 자동 재생성됩니다
- CLI는 대량 문서 일괄 처리 또는 문제 해결 시 사용

### D-6. 문제 해결

| 증상 | 확인 사항 | 해결 |
|------|----------|------|
| 백엔드 미연결 | FastAPI 실행 여부 | `python backend/main.py` 재실행, 포트 8000 확인 |
| AI 응답 오류 | Ollama 연결 | `ollama list`로 모델 확인, `OLLAMA_URL` IP 확인 |
| Ollama 연결 실패 | 방화벽 | Linux 11434 포트 개방 |
| 검색 결과 없음 | 인덱스 파일 | `data/search-index.json` 존재 여부, 재인덱싱 |
| 업로드 실패 | 권한/용량 | `contents/` 쓰기 권한, 파일 500MB 이하 |
| 업로드 변환 실패 (DRM) | COM 전처리 | `WORD_COM_PREPROCESS = False` 설정, 또는 수동 전처리 |
| 번역 실패 | pdf2zh 설치 | `pdf2zh --version` 확인, 캐시 디렉토리 확인 |
| 번역 타임아웃 | GPU 성능 | 관리자 설정에서 타임아웃 증가, GPU VRAM 확인 |
| 로그인 불가 | DB 파일 | `data/auth.db` 존재 여부, `create-admin.py` 재실행 |
| 설정 변경 미반영 | 재시작 필요 | Ollama URL/모델, CORS 등은 백엔드 재시작 필요 |

| 2 | Windows | FastAPI 프로세스 종료 (Ctrl+C) |
