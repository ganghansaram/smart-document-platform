## 메뉴 구조 관리

### 방법 1: 관리자 설정 GUI (권장)

관리자 설정 페이지의 **메뉴 관리** 탭에서 GUI로 트리 구조를 편집할 수 있습니다.

1. 좌측 메뉴 → **관리자 설정** → **메뉴 관리** 탭 클릭
2. 시스템 항목(홈/용어집/정보/대시보드/관리자설정)을 제외한 콘텐츠 트리가 표시됨

**편집 기능:**

| 기능 | 조작 |
|------|------|
| 최상위 항목 추가 | 상단 "+ 최상위 항목 추가" 버튼 |
| 하위 항목 추가 | 노드 hover → **+** 버튼 |
| 이름/URL 편집 | 노드 hover → **✎** 버튼 → 인라인 편집 (Enter로 확인) |
| 순서 이동 | 노드 hover → **▲▼** 버튼 (같은 레벨 내) |
| 삭제 | 노드 hover → **✕** 버튼 (확인 다이얼로그) |
| 저장 | 하단 "저장" 버튼 → `menu.json` 갱신 + 좌측 패널 즉시 반영 |

**삭제 시 경고**: 문서(URL)가 연결된 노드를 삭제하면 연결된 문서 경로 목록이 표시됩니다. 디스크의 파일은 삭제되지 않으며, 메뉴에서 접근할 수 없게 됩니다.

**시스템 항목 보존**: 홈/용어집/정보/대시보드/관리자설정은 서버에서 자동 보존되므로 편집/삭제할 수 없습니다.

### 방법 2: menu.json 직접 편집

#### 메뉴 파일 위치

```
data/menu.json
```

#### 메뉴 구조 이해하기

`menu.json`은 좌측 트리 메뉴의 구조를 정의하는 JSON 파일입니다.

**기본 구조**:

```json
[
  {
    "label": "메뉴 항목 이름",
    "url": "contents/파일경로.html"
  },
  {
    "label": "하위 메뉴가 있는 항목",
    "children": [
      {
        "label": "하위 항목",
        "url": "contents/subfolder/file.html"
      }
    ]
  }
]
```

#### 메뉴 항목 추가하기

**예시 1: 단일 메뉴 항목 추가**

```json
{
  "label": "새로운 문서",
  "url": "contents/new-document.html"
}
```

**예시 2: 하위 메뉴가 있는 항목 추가**

```json
{
  "label": "기술 문서",
  "children": [
    {
      "label": "설계 문서",
      "url": "contents/tech/design.html"
    },
    {
      "label": "시험 문서",
      "url": "contents/tech/testing.html"
    }
  ]
}
```

**예시 3: 3단계 깊이 메뉴**

```json
{
  "label": "시스템",
  "children": [
    {
      "label": "항전 시스템",
      "children": [
        {
          "label": "레이더",
          "url": "contents/system/avionics/radar.html"
        },
        {
          "label": "통신",
          "url": "contents/system/avionics/comm.html"
        }
      ]
    }
  ]
}
```

#### 메뉴 수정 주의사항

1. **JSON 문법 준수**: 쉼표, 중괄호, 대괄호를 정확하게 작성
2. **URL 경로**: `contents/`로 시작하는 상대 경로 사용
3. **슬래시 방향**: 항상 `/` (슬래시) 사용 (백슬래시 `\` 사용 금지)
4. **한글 사용 가능**: label은 한글로 작성 가능

#### JSON 문법 검증

메뉴 수정 후 문법 오류를 확인하려면 온라인 JSON 검증기를 사용하세요:
- JSONLint (https://jsonlint.com/)
- 또는 텍스트 에디터의 JSON 검증 기능 사용

## HTML 콘텐츠 추가

### 1. HTML 파일 생성

새로운 문서를 추가하려면 `contents/` 폴더에 HTML 파일을 생성합니다.

**기본 템플릿**:

```html
<h1>문서 제목</h1>

<h2>첫 번째 섹션</h2>
<p>본문 내용...</p>

<h3>하위 섹션</h3>
<p>상세 내용...</p>

<h2>두 번째 섹션</h2>
<p>추가 내용...</p>
```

**중요**:
- 완전한 HTML 문서(`<!DOCTYPE html>`, `<html>`, `<body>` 등)가 아닌 **본문 내용만** 작성
- 첫 번째 `<h1>` 태그가 문서 제목으로 사용됨
- `<h1>` ~ `<h6>` 태그는 우측 목차에 자동으로 표시됨
- `<h2>` 항목은 하위 헤딩(h3~h6)을 접기/펼치기 가능

### 2. 폴더 구조 생성

관련 문서를 주제별로 정리하려면 폴더를 생성합니다:

```
contents/
├── system-design/
│   ├── structure/
│   │   ├── fuselage.html
│   │   └── wing.html
│   └── avionics/
│       └── radar.html
```

### 3. menu.json에 추가

새로운 HTML 파일을 생성한 후, `data/menu.json`에 메뉴 항목을 추가합니다:

```json
{
  "label": "시스템 설계",
  "children": [
    {
      "label": "항공기 구조",
      "children": [
        {
          "label": "동체 설계",
          "url": "contents/system-design/structure/fuselage.html"
        }
      ]
    }
  ]
}
```

## Word 문서를 HTML로 변환

### 변환 전 준비: Word 스타일 적용

변환 전 Word 문서에 제목 스타일이 적용되어 있는지 확인하세요. 상세 규칙은 하단의 **팁과 모범 사례 > 문서 작성 가이드** 섹션을 참조하세요.

### 방법 1: 내장 변환기 사용 (권장)

프로젝트에 포함된 내장 변환기(`tools/converter/`)를 사용합니다. python-docx 패키지가 필요합니다.

**변환 명령어**:

```cmd
# 단일 파일 변환
python tools/converter/converter.py input.docx -o contents/output/output.html

# PDF 변환 (PyMuPDF 필요)
python tools/converter/converter.py input.pdf -o contents/output/output.html
```

**내장 변환기 장점**:
- 이미지를 `{문서명}_images/` 폴더에 자동 추출
- **이미지 비율 보존**: Word 페이지 대비 비율(%)로 출력 → 원문 레이아웃 유지
- 캡션 자동 감지 및 ID 부여 (`id="fig-1"`, `id="tbl-2"`)
- 본문 참조 텍스트를 `<a data-fig-ref>` 링크로 자동 변환
- Word SEQ 필드(캡션 번호) 자동 채번
- **수식(OMML) → MathML 자동 변환** (디스플레이/인라인 수식 모두 지원)
- **Word 도형/그리기 감지**: 추출 불가 도형 발견 시 플레이스홀더 경고 삽입
- 하이퍼링크 보존 (Word 링크 → HTML `<a href>`)
- 북마크 앵커 변환
- 각주/미주 → `<section class="footnotes">` 자동 생성
- 표 셀 병합 (`colspan`/`rowspan`) 완전 지원
- 표 셀 정렬 보존 (CENTER, RIGHT, JUSTIFY → `text-align`)
- 텍스트 색상/하이라이트 보존 (검정 제외)
- 문단 정렬: 가운데, 오른쪽, 양쪽 정렬(JUSTIFY) 지원
- 줄바꿈(`<br>`), 탭(`&emsp;`), 페이지 나누기(`<hr class="page-break">`) 처리
- 리스트 (순서/비순서, 중첩) 자동 변환
- TOC(목차) 문단 자동 스킵 (우측 On this page 패널이 대체)
- **장절번호 자동 평문화**: Word 다단계 목록 자동번호(1.1, 3.2.4 등)를 텍스트로 변환 (Word 설치 환경에서 자동 적용)

**웹 UI 업로드** (백엔드 실행 필요):
- 웹북 화면에서 메뉴 노드에 직접 Word/PDF 파일 업로드
- 변환 + menu.json 갱신 + 검색 인덱스 재생성이 자동 처리됨

### 방법 2: Pandoc 사용

외부 도구 Pandoc을 사용할 수도 있습니다 (캡션 자동 ID/참조 링크 미지원):

```cmd
pandoc input.docx -o output.html --extract-media=./images
```

### 방법 3: Word의 "웹 페이지로 저장" 기능

1. Word에서 문서 열기
2. `파일 > 다른 이름으로 저장`
3. 파일 형식: `웹 페이지, 필터링됨 (*.htm, *.html)` 선택
4. 저장

**주의**: Word가 생성한 HTML은 불필요한 스타일 태그가 많으므로 정리가 필요합니다.

### 방법 3: 수동 변환

소량의 문서는 수동으로 복사-붙여넣기하여 HTML로 작성할 수 있습니다:

1. Word 문서의 텍스트를 복사
2. 텍스트 에디터에서 HTML 태그 추가
   - 제목 → `<h1>`, `<h2>`, `<h3>`
   - 문단 → `<p>...</p>`
   - 목록 → `<ul><li>...</li></ul>`
   - 표 → `<table>...</table>`

### 수식(Equation) 지원

내장 변환기는 Word 수식(OMML)을 **MathML**로 자동 변환합니다. MathML은 최신 브라우저(Chrome, Edge, Firefox)가 네이티브 지원하므로 외부 JavaScript 라이브러리 없이 렌더링됩니다.

**지원 수식 유형:**

| 유형 | 설명 | 예시 |
|------|------|------|
| 디스플레이 수식 | 독립 블록으로 가운데 정렬 표시 | 공식, 정의식 |
| 인라인 수식 | 본문 텍스트 안에 삽입 | "변수 *x*₁은..." |

**지원 OMML 요소:**
- 분수, 아래/위첨자, 괄호, 근호(√)
- 합(∑), 적분(∫), 함수(sin, cos 등)
- 악센트(벡터 화살표 등), 행렬, 수식 배열, 극한
- 앞첨자, 그룹 문자, 테두리 박스 등

**변환 결과:**
- 디스플레이 수식: `<div class="math-display"><math display="block">...</math></div>`
- 인라인 수식: `<p>텍스트 <math>...</math> 텍스트</p>`

**주의사항:**
- 수식 편집은 **Word에서 수정 후 재변환**을 권장합니다. MathML은 기계 생성 포맷으로 사람이 직접 편집하기 어렵습니다.
- 변환 통계에 `equations` 항목으로 디스플레이 수식 개수가 표시됩니다.

### 변환 후 정리 작업

변환된 HTML은 다음 사항을 확인하고 정리합니다:

1. **불필요한 태그 제거**: `<span style="...">`, `<div class="...">` 등
2. **제목 태그 정리**: h1, h2, h3만 사용
3. **이미지 경로 확인**: 상대 경로로 수정
4. **본문만 유지**: `<html>`, `<body>` 등 불필요한 외부 태그 제거

## 검색 인덱스 업데이트

새로운 HTML 파일을 추가하거나 기존 파일을 수정한 후에는 검색 인덱스를 업데이트해야 합니다.

### 기본 사용법

```cmd
# 섹션 단위 인덱싱 + ID 주입 (권장)
python tools/build-search-index.py --inject-ids

# 벡터 인덱스 빌드 (하이브리드 검색 사용 시, Ollama bge-m3 필요)
python tools/build-vector-index.py

# 섹션 단위 인덱싱 (ID 주입 없음)
python tools/build-search-index.py

# 페이지 단위 인덱싱 (기존 방식)
python tools/build-search-index.py --page
```

> **권장**: 검색 인덱스와 벡터 인덱스를 함께 빌드하면 하이브리드 검색(키워드 + 의미 검색)이 활성화되어 AI 채팅 답변 품질이 크게 향상됩니다.

> **권장**: `--inject-ids` 옵션을 사용하면 HTML 파일에 섹션 ID가 직접 주입되어, AI 채팅의 참고 문서 링크가 정확한 섹션으로 이동합니다.

**출력 예시**:

```
============================================================
KF-21 웹북 - 검색 인덱스 생성
모드: 섹션 단위
섹션 최대 길이: 3200자
============================================================

인덱싱: contents/about.html - 7개 섹션
인덱싱: contents/home.html - 1개 섹션
인덱싱: contents/dev-overview/introduction.html - 4개 섹션

검색 인덱스 생성 완료: data/search-index.json
총 18개의 항목이 인덱싱되었습니다.
```

### 인덱싱 모드 비교

| 모드 | 명령어 | 용도 |
|------|--------|------|
| 섹션 단위 | `--section` (기본) | 긴 문서, 정밀 검색 필요 시 |
| 페이지 단위 | `--page` | 짧은 문서, 단순 검색 |

### 섹션 인덱싱 설정

스크립트 상단에서 설정값을 조정할 수 있습니다:

```python
# tools/build-search-index.py 상단

HEADING_LEVELS = [1, 2, 3]      # 분할 기준 헤딩 레벨
MAX_SECTION_LENGTH = 3200       # 섹션 최대 길이 (자)
MIN_SECTION_LENGTH = 100        # 짧은 섹션 병합 기준 (자)
```

**설정 가이드**:

| 설정 | 기본값 | 설명 | 조정 기준 |
|------|--------|------|-----------|
| `HEADING_LEVELS` | `[1,2,3]` | 분할 기준 헤딩 | 문서가 매우 길면 `[1,2,3,4]` 추가 |
| `MAX_SECTION_LENGTH` | `3200` | 섹션 최대 길이 | AI 컨텍스트 고려, 800토큰 ≈ 3200자 |
| `MIN_SECTION_LENGTH` | `100` | 병합 기준 | 너무 짧은 섹션 방지 |

**명령줄 옵션**:

```cmd
# 섹션 최대 길이 조정 (예: 2000자)
python tools/build-search-index.py --max-length 2000

# HTML 파일에 섹션 ID 직접 주입 (권장)
python tools/build-search-index.py --inject-ids

# 옵션 조합 예시
python tools/build-search-index.py --inject-ids --max-length 2000
```

**--inject-ids 옵션**:
- HTML 헤딩 태그에 `id` 속성이 없으면 자동 생성하여 주입
- AI 채팅 답변의 참고 문서 링크가 정확한 섹션으로 이동 가능
- 문서 형식이 다양하거나 예측 불가능한 경우 필수

### 문서 특성별 권장 설정

| 문서 유형 | HEADING_LEVELS | MAX_SECTION_LENGTH |
|-----------|----------------|-------------------|
| 일반 기술문서 (10~50페이지) | `[1,2,3]` | 3200 |
| 긴 매뉴얼 (100페이지+) | `[1,2,3]` | 2000 |
| 매우 긴 문서 (400페이지+) | `[1,2,3,4]` | 1600 |
| 짧은 문서 (5페이지 미만) | `--page` 모드 | - |

### 검색 인덱스 구조

`data/search-index.json` 파일 구조:

```json
[
  {
    "title": "섹션 제목",
    "url": "contents/파일경로.html",
    "path": "메뉴 > 경로 > 섹션 제목",
    "content": "섹션 본문 내용...",
    "section_id": "섹션-제목",
    "heading_level": 2
  }
]
```

- `section_id`: 해당 섹션으로 직접 이동할 수 있는 앵커 ID
- `heading_level`: 헤딩 레벨 (1=h1, 2=h2, 3=h3)

## 이미지 관리

### 이미지 파일 저장 위치

이미지는 용도에 따라 두 가지 방식으로 저장됩니다:

#### 1. 콘텐츠 이미지 (문서명_images/)

문서 본문에 사용되는 이미지는 **해당 HTML 파일과 1:1로 매핑되는 폴더**에 저장:

```
contents/dev-overview/
├── introduction.html
├── introduction_images/        # introduction.html 전용 이미지
│   ├── kf21-photo.jpg
│   └── timeline.png
├── specifications.html
└── specifications_images/      # specifications.html 전용 이미지
    └── diagram.png
```

**명명 규칙**: `{문서명}_images/`

- `introduction.html` → `introduction_images/`
- `radar.html` → `radar_images/`

**장점**:
- 문서별 이미지 소유권 명확
- 문서 삭제 시 폴더째 삭제 가능
- 파일명 충돌 없음

#### 2. UI 이미지 (css/images/)

헤더 로고, 배너 이미지 등 UI 요소는 `css/images/` 폴더에 저장:

```
css/images/
├── kai_logo.png         # 헤더 로고
├── 1-1_KF-21.jpg       # 배너 슬라이드 1
├── 1-2_KF-21.jpg       # 배너 슬라이드 2
└── 1-3_KF-21.jpg       # 배너 슬라이드 3
```

### HTML에서 이미지 참조

이미지 경로는 HTML 파일 위치 기준 **상대 경로**로 참조:

```html
<!-- contents/dev-overview/introduction.html 에서 -->
<h2>KF-21 외형</h2>
<img src="introduction_images/kf21-photo.jpg" alt="KF-21 전투기">

<h2>개발 타임라인</h2>
<img src="introduction_images/timeline.png" alt="개발 타임라인">
```

**참고**:
- 이미지 폴더는 HTML 파일과 같은 디렉토리에 위치
- Word 변환기 사용 시 자동으로 `{문서명}_images/` 폴더 생성됨
- 웹북이 콘텐츠를 로드할 때 상대 경로가 자동으로 변환되므로, HTML 파일 기준의 상대 경로로 작성하면 됨

### 이미지 크기 조절

내장 변환기는 이미지를 Word 페이지 대비 비율(%)로 출력합니다. 이미지가 너무 크거나 작게 표시되는 경우, 웹 에디터(Edit)에서 `style="width: N%"` 값을 수정하여 조절할 수 있습니다.

```html
<!-- 변환기 기본 출력 (Word 페이지 100% 폭) -->
<img src="document_images/diagram.png" alt="" style="width: 100%">

<!-- 에디터에서 80%로 축소 -->
<img src="document_images/diagram.png" alt="" style="width: 80%">

<!-- 고정 픽셀 크기로 지정 -->
<img src="document_images/diagram.png" alt="" style="width: 500px">
```

### Word 도형/그리기 누락 시

Word의 도형, 텍스트 상자, SmartArt 등은 이미지 파일이 아닌 벡터 객체로 저장되어 있어 변환 시 추출되지 않습니다. 변환기는 이를 감지하여 HTML에 경고 플레이스홀더를 삽입합니다.

**해결 방법** (Word에서 도형 → 이미지로 변환):

1. Word에서 해당 도형/그리기를 선택
2. **복사** (Ctrl+C)
3. **선택하여 붙여넣기** (Ctrl+Alt+V) → **"그림(PNG)"** 선택
4. 원래 도형을 삭제하고 붙여넣은 이미지를 사용
5. 문서를 다시 변환

### 이미지 최적화

대용량 이미지는 웹 표시용으로 최적화하는 것이 좋습니다:

- **권장 포맷**: JPG (사진), PNG (다이어그램)
- **권장 크기**: 가로 1200px 이하
- **파일 크기**: 500KB 이하

### 배너 슬라이드 추가/삭제

슬라이드 구성을 변경하려면 `js/banner.js` 파일을 수정합니다.

**파일 경로**: `js/banner.js`

```javascript
// 배너 슬라이드 설정 (이미지 + 영상 혼합 가능)
// type: 'image' | 'video'
const bannerSlides = [
    { type: 'video', src: 'css/images/kf21_video_sample.mp4' },
    { type: 'image', src: 'css/images/1-1_KF-21.jpg' },
    { type: 'image', src: 'css/images/1-2_KF-21.jpg' },
    { type: 'image', src: 'css/images/1-3_KF-21.jpg' }
];
```

**슬라이드 추가 예시**:
1. 새 파일을 `css/images/` 폴더에 저장
2. `banner.js`의 `bannerSlides` 배열에 `{ type: 'image', src: '경로' }` 또는 `{ type: 'video', src: '경로' }` 추가
3. 저장 후 브라우저 새로고침

**슬라이드 삭제 예시**:
1. `bannerSlides` 배열에서 해당 항목 삭제
2. (선택사항) `css/images/` 폴더에서 파일 삭제

### 배너 슬라이드 속도 변경

`js/banner.js` 파일에서 속도를 조정할 수 있습니다:

```javascript
// 슬라이드쇼 설정
const slideConfig = {
    interval: 4000,      // 4초마다 전환 (밀리초 단위)
    fadeDuration: 1000   // 1초 페이드 효과
};
```

- **interval**: 슬라이드 전환 간격 (밀리초)
  - 예: 10초로 변경 → `interval: 10000`
- **fadeDuration**: 페이드 효과 시간 (밀리초)
  - 예: 0.5초로 변경 → `fadeDuration: 500`

### 홈페이지 섹션 링크 자동 생성

홈페이지의 섹션 링크는 `data/menu.json`에서 자동으로 생성됩니다.

**동작 방식**:
- `menu.json`의 1레벨 항목 중 `children`이 있는 항목만 표시
- 각 섹션의 첫 번째 하위 페이지로 자동 연결
- 섹션 설명은 `js/banner.js`의 `descriptions` 객체에서 관리

**섹션 설명 수정**:

`js/banner.js` 파일 수정:

```javascript
const descriptions = {
    '개발 개요': '프로그램 소개, 개발 히스토리, 주요 제원',
    '시스템 설계': '항공기 구조, 항전, 추진 시스템',
    '시험 평가': '지상 시험 및 비행 시험',
    '기술 문서': '설계, 운용 문서 및 보고서',
    '참고 자료': '규격, 문헌, 용어 정의',
    '새 섹션': '새 섹션에 대한 설명'  // 추가
};
```

**주의**: 섹션 이름(`'개발 개요'`)은 `menu.json`의 `label`과 정확히 일치해야 합니다.

## AI 채팅 설정

웹북에 내장된 AI 채팅 기능은 문서 내용에 대한 질문에 답변합니다.

### 동작 모드

AI 채팅은 두 가지 모드로 동작합니다:

| 모드 | 설명 | 필요 서버 |
|------|------|-----------|
| **백엔드 모드** (권장) | FastAPI 백엔드를 통해 Ollama 호출 | Tomcat + 백엔드 + Ollama |
| **직접 호출 모드** | 브라우저에서 Ollama 직접 호출 | Tomcat + Ollama |

**백엔드 모드 장점:**
- 하이브리드 검색(키워드 + 벡터), 리랭킹, 멀티턴 대화 등 고급 기능 활용
- Ollama 서버 주소를 백엔드에서 관리 (프론트엔드 수정 불필요)

### AI 설정 파일

`js/config.js` 파일에서 환경에 맞게 설정:

```javascript
const AI_CONFIG = {
    // === 챗봇 활성화 ===
    enabled: true,                          // false: 챗봇 아이콘 숨김

    // === 모드 선택 ===
    useBackend: true,                       // true: 백엔드 모드, false: 직접 호출 모드
    backendUrl: 'http://localhost:8000',    // 백엔드 서버 주소

    // === 직접 호출 모드 설정 (useBackend: false 일 때만 사용) ===
    ollamaUrl: 'http://localhost:11434',    // Ollama 서버 주소
    model: 'gemma3:4b',                     // 사용할 모델

    // === 공통 설정 ===
    maxContextLength: 8000,  // AI에 전달할 최대 문자 수 (백엔드 기준)
    maxSearchResults: 5      // 검색 결과 최대 개수
};
```

### 환경별 설정 예시

#### 개발 환경 (로컬 PC)
```javascript
useBackend: true,
backendUrl: 'http://localhost:8000',
```

#### 운영 환경 (폐쇄망) - 백엔드 모드

### 콘텐츠 표시 설정

`js/config.js`의 `DISPLAY_CONFIG`로 테이블 스타일을 전환합니다 (변환 불필요, 새로고침만):

```javascript
const DISPLAY_CONFIG = {
    platformName: 'WebBook',   // 플랫폼 이름 (고정)
```

| 스타일 | 설명 |
|--------|------|
| `bordered` | 모든 셀 테두리 + 네이비 헤더 배경 (기본값) |
| `simple` | 가로선만 + 볼드 헤더 |

## 문서 편집기 설정

웹북에서 직접 문서를 편집하고 저장할 수 있는 기능입니다.

### 편집기 활성화

`js/config.js` 파일에서 설정:

```javascript
const EDITOR_CONFIG = {
    // 편집 기능 활성화
    enabled: true,                          // true: 편집 버튼 표시

    // 인증 설정
    requireAuth: true,                      // true: admin 로그인 필요

    // 백엔드 설정
    backendUrl: 'http://localhost:8000',    // 저장 API 서버

    // 편집기 설정
    autoSaveInterval: 30000,                // 자동 저장 간격 (ms), 0이면 비활성화
    createBackup: true,                     // 저장 전 .bak 파일 생성
};
```

### 설정 옵션

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `enabled` | `false` | 편집 버튼 표시 여부 |
| `requireAuth` | `true` | admin 로그인 필요 여부 |
| `backendUrl` | `http://localhost:8000` | 저장 API 서버 주소 |
| `autoSaveInterval` | `30000` | 자동 저장 간격 (밀리초), 0이면 비활성화 |
| `createBackup` | `true` | 저장 전 백업 파일 자동 생성 |

### 편집기 사용 방법

1. **편집 버튼**: 상단 네비게이션에 "Edit" 버튼 (문서 페이지에서만 표시)
2. **편집 모드**: 버튼 클릭 시 편집기 모달 열림
3. **저장**: "Save" 버튼 또는 `Ctrl+S`로 저장
4. **취소**: "Cancel" 버튼 또는 `ESC`로 편집 취소

### 편집기 특징 (Monaco Editor)

편집기는 **Monaco Editor** (VS Code 기반)를 사용합니다:

```
┌─────────────────────────────────────────────────────────┐
│  Document Editor                               [Save][X]│
├───────────────────────┬─────────────────────────────────┤
│  HTML Source          │  Preview                        │
├───────────────────────┼─────────────────────────────────┤
│  <h1>제목</h1>        │  제목                           │
│  <p>본문...</p>       │  본문...                        │
│                       │                                 │
│  (코드 편집)          │  (실시간 미리보기)              │
└───────────────────────┴─────────────────────────────────┘
```

**주요 기능:**
- **HTML 소스 편집**: 문법 하이라이팅, 자동완성
- **실시간 미리보기**: 우측에서 결과 즉시 확인
- **커서 위치 하이라이트**: 편집 중인 요소가 미리보기에서 강조
- **HTML 100% 보존**: 의도하지 않은 스타일 변경 없음
- **백업 자동 생성**: 저장 전 원본 파일 백업

### 왜 소스 에디터인가?

WYSIWYG 에디터(Toast UI, TinyMCE 등)는 HTML 구조를 변경하는 문제가 있습니다:
- 테이블 헤더(`<th>`)가 일반 셀(`<td>`)로 변경
- 인라인 스타일 제거/추가
- 의도하지 않은 포맷 변경

Monaco 소스 에디터는 **수정한 부분만 정확히 변경**됩니다.

### 백업 파일

저장 시 백업 파일이 `backups/` 폴더에 자동 생성됩니다:

```
backups/

├── specifications_20250209_150123.bak
└── introduction_20250209_151234_before_restore.bak
```

**파일명 형식**: `{문서명}_{YYYYMMDD}_{HHMMSS}.bak`

### 편집 가능 문서

다음 문서는 편집이 **비활성화**됩니다:
- `home.html` (홈페이지)
- `about.html` (소개 페이지)

### 백엔드 요구사항

편집 기능을 사용하려면 **백엔드 서버가 실행 중**이어야 합니다:

```cmd
cd backend

```

> **참고**: AI 채팅과 문서 편집은 **동일한 백엔드**를 사용합니다.

- 경로 탈출 방지: `contents/` 폴더 외부 접근 차단
- 인증 연동: `requireAuth: true` 설정 시 admin 로그인 필요 (기본값)

> **참고**: 편집 후 검색 인덱스도 업데이트하려면 `python tools/build-search-index.py --inject-ids`를 다시 실행하세요.

### 향후 개선 사항

- 문서 버전 히스토리 UI
- 동시 편집 충돌 방지

## 그림/표 참조 팝업

문서 본문에서 "Figure 1", "그림 2" 같은 참조 텍스트를 클릭하면 해당 그림이나 표를 팝업으로 바로 확인할 수 있습니다.

### 기능 동작

1. 본문에서 파란색 점선 밑줄이 있는 참조 텍스트 클릭
2. 해당 그림/표가 모달 팝업으로 표시
3. ESC 키, 배경 클릭, 또는 X 버튼으로 닫기
4. "원본 위치로 이동" 버튼으로 해당 그림/표 위치로 스크롤

### 에디터로 수작업 추가하기

변환기에서 자동 적용되지 않는 문서는 에디터로 직접 마크업을 추가할 수 있습니다.

#### 1단계: 그림/표 캡션에 ID 부여

캡션 `<p>` 태그에 `id` 속성을 추가합니다:

```html
<!-- 수정 전 -->
<p><strong>Figure 1 – 시스템 구성도</strong></p>

<!-- 수정 후 -->
<p id="fig-1"><strong>Figure 1 – 시스템 구성도</strong></p>
```

#### 2단계: 본문 참조 텍스트에 링크 추가

본문에서 그림/표를 언급하는 텍스트를 `<a data-fig-ref>` 태그로 감쌉니다:

```html
<!-- 수정 전 -->
<p>시스템 구성은 Figure 1에 나타나 있다.</p>

<!-- 수정 후 -->
<p>시스템 구성은 <a data-fig-ref="fig-1">Figure 1</a>에 나타나 있다.</p>
```

#### ID 규칙

| 유형 | 접두어 | 예시 |
|------|--------|------|
| 그림 | `fig-` | `fig-1`, `fig-2` |
| 표 | `tbl-` | `tbl-1`, `tbl-2` |

> **참고**: `data-fig-ref` 값은 캡션의 `id` 값과 정확히 일치해야 합니다.

### 변환기 자동 적용

내장 변환기(`tools/converter/converter.py`)는 Word 문서 변환 시 다음을 자동으로 처리합니다:

1. **SEQ 필드 자동 채번**: Word에서 필드 갱신(Ctrl+A → F9) 없이 저장해도, 빈 캐시 값을 자동으로 채번
2. **캡션 ID 부여**: "Figure 1 – 제목", "표 3-1: 설명" 등의 캡션을 감지하여 `id="fig-1"`, `id="tbl-3-1"` 자동 추가
3. **본문 참조 링크**: 본문에서 "Figure 1", "그림 2", "표 3" 등의 참조 텍스트를 `<a data-fig-ref="fig-1">` 링크로 자동 변환

따라서 변환기를 통해 생성된 HTML 문서는 별도 수작업 없이 팝업 기능이 동작합니다.

> **캡션 감지 조건**: 캡션으로 인식되려면 번호 뒤에 구분자(`: – — - .`)가 있어야 합니다.
> 예: "Figure 1 – Title" (O), "Figure 1을 보면" (X, 본문 참조로 처리)

기술 상세는 [05-ARCHITECTURE.md](05-ARCHITECTURE.md#12-문서-변환-파이프라인)를 참조하세요.

## 다크/라이트 모드

헤더 우측 nav 영역의 해/달 아이콘을 클릭하여 다크/라이트 테마를 전환할 수 있습니다.

### 동작 방식

- **기본값**: 라이트 모드
- **전환**: 달 아이콘(🌙) 클릭 → 다크 모드, 해 아이콘(☀) 클릭 → 라이트 모드
- **저장**: 선택한 테마는 브라우저 localStorage에 저장되어 새로고침/재방문 시 유지
- **인쇄**: 다크 모드에서도 인쇄 시 라이트 색상이 자동 적용

### 다크 모드 이미지 처리

문서 내 이미지는 다크 모드에서 흰색 배경이 자동 적용됩니다. 투명 배경 PNG 다이어그램이 어두운 배경에서 보이지 않는 문제를 방지합니다. 홈 배너 이미지에는 적용되지 않습니다.

### 설정 변경 (config.js)

별도 설정 항목 없이 동작합니다. 테마 선택은 사용자별 브라우저에 저장됩니다.

## 북마크

자주 참조하는 섹션을 저장하고 빠르게 이동할 수 있는 기능입니다. 데이터는 브라우저 localStorage에 저장되므로 서버 없이 동작합니다.

### 북마크 추가/제거

1. 문서의 헤딩(h1~h4) 위에 마우스를 올리면 텍스트 옆에 ☆ 아이콘이 표시됩니다
2. ☆ 클릭 → ★로 변경 (북마크 추가)
3. ★ 다시 클릭 → ☆로 복귀 (북마크 제거)

### 북마크 목록 보기

1. 상단 헤더의 **Bookmarks** 메뉴를 클릭
2. 저장된 북마크가 문서별로 그룹핑되어 표시
3. 항목 클릭 → 해당 섹션으로 이동 (같은 문서는 스크롤, 다른 문서는 로드 후 스크롤)
4. 항목 호버 시 나타나는 × 버튼으로 개별 삭제
5. **Clear All** 버튼으로 전체 삭제 (확인 후)

### 특징

- 새로고침/재방문 시에도 북마크 유지 (localStorage)
- 문서 간 이동 지원 (다른 문서의 북마크 클릭 시 해당 문서 로드 → 섹션 스크롤)
- 다크 모드에서 정상 표시
- 북마크가 없을 때 빈 상태 안내 메시지 표시

## 항공 용어집

26,000+ 항공 용어(약어, 영문 풀네임, 한국어)를 검색하고 탐색할 수 있는 기능입니다. 좌측 트리 메뉴의 "용어집" 항목에서 접근합니다.

### 용어집 페이지

- **A-Z 카드 그리드**: 초기 화면에서 알파벳별 용어 수와 미리보기 표시
- **알파벳 필터**: 상단 바에서 알파벳 클릭 시 해당 그룹 테이블 표시
- **실시간 검색**: 약어, 영문, 한국어로 검색 (300ms 디바운스)
- **우측 패널**: A-Z 퀵링크로 빠른 알파벳 전환

### 본문 약어 자동 인식

문서를 열면 본문에 등장하는 용어집 약어에 **점선 밑줄**이 자동 표시됩니다.

- 약어 클릭 → 풀네임/한국어 해설 팝업
- "용어집에서 보기" 링크로 용어집 페이지 이동
- `content-visibility: auto`와 호환되는 지연 처리 방식 (성능 영향 없음)

### 통합 검색 연동

상단 검색(돋보기 아이콘)에서 검색하면 **용어집 매칭 결과가 상단 그룹**으로 표시됩니다.

- 최대 3건 미리보기 + "더 보기" 링크
- 개별 용어 클릭 → 용어집 페이지에서 해당 용어 필터링

### 용어 데이터 관리

용어 데이터는 CSV 파일로 관리하고, 변환 스크립트로 JSON으로 변환합니다.

**파일 위치:**

```
data/glossary.csv       # CSV 원본 (관리용)
data/glossary.json      # 웹북이 사용하는 JSON (자동 생성)
tools/import-glossary.py # CSV → JSON 변환 스크립트
```

**CSV 양식** (UTF-8 with BOM 권장):

```csv
abbr,en,ko
AAA,Anti-Aircraft Artillery,대공포
FCS,Fire Control System,사격통제체계
RWR,Radar Warning Receiver,레이더경보수신기
```

**변환 명령어:**

```cmd
# 기본: data/glossary.csv → data/glossary.json
python tools/import-glossary.py

# 경로 지정
python tools/import-glossary.py path/to/custom.csv -o data/glossary.json
```

**변환 기능:**
- 헤더 검증 (`abbr`, `en` 필수)
- 중복 항목 감지 및 경고
- `abbr` 기준 알파벳 오름차순 자동 정렬
- 빈 행 자동 스킵

### 용어 추가/수정 절차

1. `data/glossary.csv`를 엑셀(또는 한셀)로 편집
2. 저장 (UTF-8 with BOM 형식)
3. 변환 실행: `python tools/import-glossary.py`
4. 브라우저 새로고침

> **참고**: `glossary.json`을 직접 편집하지 마세요. 항상 CSV를 수정하고 변환 스크립트를 통해 JSON을 생성합니다.

## 관리자 인증

v5.4부터 3단계 역할 기반 접근 제어(RBAC)를 지원합니다.

### 역할 체계

| 역할 | 레벨 | 권한 |
|------|------|------|
| **viewer** | 1 | 콘텐츠 열람, 검색, AI 채팅, 북마크, 테마, 용어집 |
| **editor** | 2 | viewer + 문서 편집/저장 |
| **admin** | 3 | editor + 업로드, 인덱싱, 사용자 관리, 설정 관리, 통계 대시보드 |

### 로그인 페이지

`login.html`은 독립된 로그인 페이지입니다.

- `AUTH_CONFIG.loginRequired = true`이면 미로그인 시 자동으로 `login.html`로 이동
- 로그인 성공 시 체크마크 피드백 → 부드러운 fade-out → launcher.html 전환
- 이미 로그인된 상태면 자동으로 launcher.html로 리다이렉트

### 최초 admin 계정 생성

```bash
python tools/create-admin.py
```

서버 없이 직접 DB(`data/auth.db`)에 계정을 생성하므로, 비상 복구 시에도 사용 가능합니다.

### 로그인/로그아웃

- 헤더 네비게이션의 **Login** 클릭 → 로그인 모달 또는 login.html로 이동
- 로그인 후 헤더에 사용자명, 역할 배지, **Logout** 링크가 표시됩니다

### 사용자 관리 (admin 전용)

관리자 설정 페이지(`admin.html`)의 **계정 관리** 탭에서 사용자를 관리합니다:

- 사용자 추가: Username + Password + 역할(viewer/editor/admin) 선택 후 **추가**
- 비밀번호/역할 변경: 테이블의 **Edit** 클릭
- 사용자 삭제: 테이블의 **Delete** 클릭 (본인 삭제 불가)

### 설정

`js/config.js`:

```javascript
const AUTH_CONFIG = {
    enabled: true,                    // false: 인증 비활성화
    loginRequired: true,              // true: 미로그인 시 login.html로 이동
    backendUrl: 'http://localhost:8000',
};
```

> **참고**: `loginRequired`를 `false`로 설정하면 로그인 없이 열람할 수 있습니다. 관리 기능은 여전히 로그인이 필요합니다.

|---|---|---|
| **공통** | Ollama, 검색, RAG, 세션, 보안, 업로드/변환 | 일부 |
| **Explorer** | 사이트 타이틀, 테이블 스타일, 챗봇, 에디터, 메뉴 관리 | 즉시 |
| **Translator** | 번역 모델, PDF 크기, 타임아웃, AI 선택 프롬프트 | 즉시 |

### 동작 방식

- **저장**: 설정값이 `data/settings.json`에 저장되고 백엔드 config에 즉시 반영
- **재시작 필요 항목**: 저장 시 재시작이 필요한 항목 목록이 알림으로 표시됨
- **초기화**: "초기화" 버튼으로 모든 설정을 기본값으로 복원 (settings.json 삭제)
- **프론트엔드 동기화**: `loadRuntimeSettings()`가 `/api/settings/public`에서 설정을 가져와 CONFIG 객체에 오버라이드

### 사이트 타이틀 변경

"화면 / 에디터" 탭에서 **사이트 타이틀** 필드를 수정하면:
- 헤더의 `<h1>` 텍스트
- 브라우저 탭 `<title>`
- 로그인 페이지 브랜드 타이틀

이 모두 동적으로 변경됩니다.

### Translator AI 선택 설정

Translator 탭의 **AI 선택** 서브탭에서 텍스트 선택 번역/요약 기능을 설정합니다:

| 설정 | 기본값 | 설명 |
|------|--------|------|
| 번역 프롬프트 | (한국어 번역 + 기술 용어 영문 병기) | AI 번역 시스템 프롬프트 |
| 요약 프롬프트 | (3문장 이내 한국어 요약) | AI 요약 시스템 프롬프트 |
| 타임아웃 (초) | 30 | AI 응답 대기 시간 (5~120초) |

이 설정은 재시작 없이 즉시 적용됩니다.

## 접속 통계 (Analytics)

admin 권한 사용자가 관리자 설정 페이지의 **대시보드** 탭에서 접속 현황과 사용 통계를 확인할 수 있습니다.

### 대시보드 항목

- **현재 접속자**: 실시간 접속 수 (클릭 시 IP/사용자 목록 모달)

로그인한 사용자는 60초마다 서버에 활동 정보(username)를 전송합니다. 이 데이터를 기반으로 실시간 접속자를 판별합니다.

## 키보드 단축키

웹북에서 키보드 단축키를 사용하여 빠르게 탐색할 수 있습니다. `?` 키를 누르면 단축키 도움말 모달이 표시됩니다.

### 단축키 목록

| 키 | 기능 |
|----|------|
| `?` | 단축키 도움말 열기/닫기 |
| `/` | 검색 열기 |
| `Ctrl+K` | 검색 열기 |
| `Esc` | 모달/오버레이 닫기 |
| `←` | 이전 문서로 이동 |
| `→` | 다음 문서로 이동 |
| `H` | 홈으로 이동 |
| `B` | 북마크 열기 |

### 동작 규칙

- **입력 필드 활성 시 무시**: 검색창, 텍스트 입력 등에 포커스가 있으면 단축키가 동작하지 않음
- **오버레이 열린 상태**: 검색, 북마크 등 오버레이가 열려있으면 단축키 무시 (Esc로 닫기만 가능)
- **문서 끝 도달 알림**: `←`/`→`로 첫 번째/마지막 문서에 도달하면 토스트 알림 표시

## 팁과 모범 사례

### 1. 파일 및 폴더 이름 규칙

- **소문자 사용**: `system-design.html` (권장) vs `System-Design.html` (비권장)
- **하이픈 사용**: `flight-test.html` (권장) vs `flight_test.html` (혼용 가능)
- **한글 사용 자제**: 파일명은 영문으로, 메뉴 label만 한글 사용
- **공백 사용 금지**: `flight test.html` (절대 금지)

### 2. 문서 작성 가이드

#### 필수: Word 제목 스타일 적용

변환기는 Word의 **스타일 이름**을 기준으로 헤딩 태그를 생성합니다. 폰트 크기나 굵기만으로 제목을 구분한 문서는 모두 본문(`<p>`)으로 변환되어 **목차, 섹션 래핑, 검색 인덱싱이 동작하지 않습니다**.

1. Word에서 제목으로 사용할 텍스트를 선택
2. `홈 > 스타일`에서 "제목 1", "제목 2", "제목 3" 등을 적용
3. 저장 후 변환

| Word 스타일 | 변환 결과 | 웹북 동작 |
|------------|----------|----------|
| 제목 1 / Heading 1 | `<h1>` | 문서 제목, 섹션 래핑 기준, 목차 최상위 |
| 제목 2 / Heading 2 | `<h2>` | 주요 섹션 단위, 목차 접기/펼치기 기준 |
| 제목 3~6 / Heading 3~6 | `<h3>`~`<h6>` | 하위 항목, 목차에 중첩 표시 |
| 스타일 없음 (굵기/크기만) | `<p>` | 목차 미표시, 섹션 래핑 안 됨 |

> **참고**: 이 방식은 Pandoc, LibreOffice 등 모든 주요 변환 도구의 공통 요구사항입니다.

#### 장절번호 자동 처리

Word의 다단계 목록 자동번호(1, 1.1, 1.1.1 등)는 변환 시 자동으로 텍스트로 평문화됩니다. 수동으로 타이핑한 번호와 자동번호가 중복되지 않도록, 둘 중 하나만 사용하세요.

- **자동번호 사용 (권장)**: Word 다단계 목록으로 번호 관리 → 변환 시 자동 추출
- **수동 번호 사용**: 헤딩 텍스트에 직접 "1.1 제목" 입력 → 변환 시 그대로 유지
- **Word 미설치 환경**: 자동번호 평문화가 건너뛰어지므로, 수동 번호를 사용하거나 Word가 설치된 환경에서 업로드

#### 권장: 구조 설계

- **h1은 문서당 한 번**: 문서 제목으로 사용
- **h2를 주요 섹션 단위로**: 우측 목차에서 h2 아래의 h3~h6이 접기/펼치기됨
- **h1 ~ h6 모두 목차에 표시**: 논리적 계층을 잘 설계하면 탐색이 쉬워짐
- **표 활용**: 데이터는 표로 정리 (병합, 정렬, 스타일 프리셋 모두 지원)
- **이미지 크기**: Word에서 지정한 배치 크기가 그대로 반영됨

### 3. 메뉴 구조 설계

- **깊이 제한**: 3단계 이내로 유지 (더 깊으면 탐색이 어려움)
- **논리적 그룹**: 관련 문서를 같은 카테고리로 묶기
- **균형**: 한 카테고리에 너무 많은 항목 배치 금지 (5~7개 권장)

### 4. 대용량 문서 최적화

대용량 문서(수백 페이지, 다수 이미지)는 다음 사항을 참고하세요:

- **자동 최적화**: 콘텐츠 로드 시 `content-visibility: auto` 섹션 래핑이 자동 적용되어 렌더링 성능이 개선됩니다
- **h1/h2 구조 활용**: 섹션 래핑이 h1/h2 기준으로 수행되므로, h1과 h2를 적절히 사용하면 최적화 효과가 높아집니다
- **이미지 권장 크기**: 가로 1200px 이하, 파일 당 500KB 이하
- **문서 분할 권장**: 400페이지 이상의 문서는 챕터 단위로 분할하면 로딩 속도와 AI 검색 품질이 모두 향상됩니다

### 5. 검색 인덱스 관리

콘텐츠 변경 시 검색 인덱스와 벡터 인덱스를 함께 빌드하는 것을 권장합니다:

```cmd
:: 검색 인덱스 + 벡터 인덱스 함께 빌드
python tools/build-search-index.py --inject-ids
python tools/build-vector-index.py
```

> **참고:** 웹 UI에서 문서를 업로드하면 검색 인덱스와 벡터 인덱스가 자동으로 갱신됩니다.

### 6. 정기적인 백업

중요한 변경 전에는 다음 파일을 백업하세요:
- `data/menu.json`
- `data/search-index.json`
- `contents/` 폴더 전체

### 7. 버전 관리

문서 변경 이력을 추적하려면:
- 파일명에 버전 번호 또는 날짜 추가 (예: `manual-v2.html`, `guide-20250124.html`)
- 또는 Git과 같은 버전 관리 시스템 사용

## 문제 해결

### 메뉴에 항목이 표시되지 않음

- `menu.json` 문법 오류 확인
- 브라우저 캐시 삭제 (Ctrl+F5)

### 검색 결과가 나오지 않음

- 검색 인덱스 재생성 (`build-search-index.py` 실행)
- `data/search-index.json` 파일 존재 확인

### 이미지가 표시되지 않음

- 이미지 파일 경로 확인 (HTML 파일 기준 상대 경로로 작성되었는지 확인)
- 이미지 파일명 대소문자 확인 (Linux는 대소문자 구분)
- 웹북은 상대 경로를 자동 변환하므로, HTML 파일을 직접 열어서 이미지가 보이면 웹북에서도 정상 표시됨

### AI 채팅이 작동하지 않음

**백엔드 모드 (useBackend: true):**

- **"백엔드 서버에 연결할 수 없습니다"**
  - 백엔드 실행 확인: `python backend/main.py`
  - 포트 확인: `netstat -ano | findstr :8000`
  - `js/config.js`의 `backendUrl` 주소 확인

- **백엔드는 되는데 AI 응답 오류**
  - `backend/config.py`의 `OLLAMA_URL` 확인
  - Ollama 서버 연결 테스트: `curl http://<ollama-ip>:11434/api/tags`

**직접 호출 모드 (useBackend: false):**

- **"Ollama 서버에 연결할 수 없습니다"**
  - Ollama가 실행 중인지 확인: `ollama list`
  - Ollama 재시작: `ollama serve` (Linux) 또는 앱 재시작 (Windows)
  - `js/config.js`의 `ollamaUrl` 주소 확인

- **"모델을 찾을 수 없습니다"**
  - 모델 설치 확인: `ollama list`
  - 모델 설치: `ollama pull gemma3:4b`
  - `js/config.js`의 `model` 이름이 설치된 모델과 일치하는지 확인

**공통:**

- **응답이 부정확함**
  - 질문을 더 구체적으로 작성
  - 관련 문서 페이지에서 질문 (현재 페이지 내용이 우선 참조됨)
  - 대용량 문서는 분할 고려
